Sudonit · near-eye UX prototypes

Four interactive concepts, each rendered inside a simulated monocular, monochrome, low-resolution near-eye display (the real V1/V2 constraint — see docs/DISPLAY_EVALUATION.md). They are intentionally austere: if a design needs full colour or a full screen, it doesn't belong on these glasses. Keyboard-interactive; no build step.

▶ Virtual device — primary product-validation tool

A realistic, runnable Sudonit you can live in before hardware: full flows for all 8 apps, a real connectivity model (phone / internet / AI as independent axes) with graceful degradation, first-run setup (Wi-Fi + AI provider + tutorial), scripted real-world scenarios A–E, and live interaction metrics. Flip the phone link off and watch it stay useful. The goal: know if Sudonit feels useful or frustrating now.

Reviews from living in it: DESIGN_REVIEW.md (brutal UX critique + metrics) · FUTURE_HARDWARE_IMPACT.md (decisions across hardware tiers).

The four static concepts below explore the look of individual surfaces; the virtual device above exercises the whole experience.

A · Minimal AI assistant

Space = ask · N = next

Press, look, hear. One-line glance echo of a primarily-spoken answer. The safest, most shippable concept — closest to the MVP.

B · AR-style HUD

Space = identify · ←/→ = focus

Loose, honest "AR": parked labels + callouts, not pixel-locked anchors — because monocular monochrome can't do registered AR.

C · Navigation

Space = walk · R = restart

Big shape-coded turn arrow + distance + street. The use case where a tiny monochrome HUD is genuinely, immediately useful.

D · Accessibility

Space = play · +/− size · C = contrast

Live captions with user-controlled text size and high-contrast mode. The most humane, defensible reason for a camera+mic on your face.

These prototypes inform docs/DISPLAY_ARCHITECTURE.md §5 (the four content renderers map 1:1 to these concepts). They run entirely in the browser so UX can be iterated with zero hardware — the same pre-hardware discipline as the rest of the project.
